Seminário Position Display

No desenvolvimento web, o CSS é responsável por organizar e posicionar os elementos. As propriedades display e position são fundamentais: a primeira define como os elementos aparecem no layout, enquanto a segunda controla sua posição na página. Este trabalho busca explicar essas propriedades e seu funcionamento na prática em HTML.

O CSS display é uma propriedade utilizada para organizar os elementos na página HTML. Ela pode conter diferentes valores, entre eles o flex e o grid, que possibilitam a criação de layouts responsivos.

Valores display:

Inline

Apresenta o elemento como conteúdo corrido,
inserido organicamente junto ao texto ou outros elementos,
ocupando apenas sua própria dimensão na tela, sem que propriedades
como width ou height provocam qualquer efeito.

Valores display:

Block

Apresenta o elemento como um bloco, inserido separadamente do conteúdo
e ocupado (até que se altere) 100% do espaço disponível na tela. None: não exibe o elemento na tela.

Valores display:

Inline-Block

Apresenta o elemento como conteúdo corrido, inserido organicamente
junto ao texto ou outros elementos, mas permite que propriedades como
width ou height o modifiquem como se fosse um tipo block

Valores display:

Grid

Apresenta o elemento como um contêiner de grid,
tipologia disponível apenas no HTML5 e em alguns navegadores.

Valores display:

Flex

Define o elemento como um contêiner flexível,
facilitando o alinhamento de itens filhos.

Valores position:

Static

O valor padrão de position é static, que mantém o elemento
no fluxo normal da página e não permite usar top, right, bottom ou
left para posicionamento. Outros valores, como relative e fixed, permitem alterar a posição do elemento.

Valores position:

Relative

O valor relative significa "relativo". Um elemento
com "position: relative" também é posicionado de acordo com o fluxo
normal da página, mas agora podemos usar as propriedades top, right, bottom e left para
ajustar sua posição relativa à sua posição original. Exemplo: .elemento

Exemplo -> position: relative;
top: 10px; /* Move o elemento 10 pixels para baixo */
left: 20px; /* Move o elemento 20 pixels para a direita */

Valores position:

Absolute

O absolute position é a ferramenta que remove o elemento do fluxo normal,
não afeta e também não é afetada pela posição dos outros elementos.
Diferente dos outros elementos no body, você pode posicionar qualquer elemento absoluto
dentro de um elemento se o mesmo estiver definido como algo diferente de static (O elemento que é posicionado conforme
o fluxo normal do html). Para definir o posicionameto é possível utilizar top left, right e bottom.

Valores position:

Fixed

O fixed position é, de maneira simples, elementos posicionados em relação a área de visualização
ou a janela do navegador, ou seja, ao rolar a página para baixo, os elementos fixos permanecem
fixos em suas posições. Essa configuração de posição é frequentemente usada para navegações fixas.
Vale salientar que, um elemento fixo se comporta de forma relativa ao fluxo do documento até atingir
uma posição de rolagem definida. Nesse ponto, ele passa a se comportar como um elemento fixo dentro de seu elemento pai direto.
Assim que o elemento fixo atinge a parte inferior do elemento pai, ele para de rolar.

Valores position:

Sticky

Para falar do sticky position é preciso também falar do fixed e do relative, pois o sticky é a junção dos dois elementos,
ele funciona como um fixed, fixa alguma imagem, texto ou elemento, porém, até um ponto em que o desenvolvedor usando o comando top: 0 estipula,
o que torna o elemento fixo até o final do contêiner, travando quando atinge o ponto definido.

Temas

Display: Block

Significa que o elemento é exibido como um bloco, assim como parágrafos e cabeçalhos sempre foram. O elemento gera uma caixa de bloqueio, produzindo quebras de
linha antes e depois do elemento no fluxo normal.

Exemplo Display: Block

Display: inline

Significa que o elemento é exibido em linha, dentro do bloco atual, na mesma linha. O elemento gera uma ou mais caixas embutidas que não geram quebras de linha antes ou depois de si mesmas.

Exemplo Display: Inline

Display: inline-block

Significa que os elementos fiquem lado a lado na mesma linha, mas podem ser configurados como uma imagem. Um bloco inline é inserido em linha (ou seja, na mesma linha que o conteúdo adjacente), mas se comporta como um bloco.

display: Flex

Significa que os elementos dentro do flex podem ser alinhados. Torna o elemento um flex container automaticamente transformando todos os seus “filhos” diretos em flex itens.

Significa que o display: inline-flex não faz com que os itens flexíveis sejam exibidos em linha. Faz com que o contêiner flexível seja exibido em linha. Não há diferença visível no site.

Display: Grid

Grid é o ponto de partida para criar um grid container. Ela transforma um elemento HTML em um contêiner de grade, onde você pode organizar os elementos filhos em linhas e colunas.

Exemplo Display: Grid

Exemplo pt2

Display: Inline-grid

Faz com que o container se comporte como um elemento inline, ocupando apenas o espaço necessário e permitindo outros elementos ao seu lado.

Exemplo Display: Inline-grid

Exemplo pt2

Display: Flow-root

Cria um novo contexto de formatação de bloco (BFC — Block Formatting Context). Isso faz com que o elemento passe a gerenciar seu próprio fluxo de layout. Um elemento com flow-root contém elementos com float, evitando que eles “escapem” do container. Também impede o colapso de margens com elementos externos.

Exemplo Display: flow-root

Exemplo pt2

Display: Table

Faz com que o elemento se comporte como uma tabela no layout do CSS, mesmo que não seja uma tag . Permite organizar elementos em linhas e células, usando valores como table-row e table-cell. O layout segue o algoritmo de tabela, ajustando automaticamente largura e alinhamento das células.

Exemplo Display: table

Display: Block Flow

Define o elemento como um container de bloco que participa do fluxo normal do documento. O elemento ocupa toda a largura disponível da linha e inicia automaticamente uma nova linha antes e depois dele. Os elementos filhos são organizados no fluxo de bloco padrão (flow layout), ou seja, posicionados verticalmente um abaixo do outro, seguindo a ordem do HTML
